# Patterns

## Naming Conventions
- Files: kebab-case (`security-guard.py`, `dev-browser`)
- Functions: camelCase (`findProjectRoot`, `waitForPageLoad`)
- Types/Interfaces: PascalCase (`DevBrowserServer`, `ServeOptions`)
- Constants: UPPER_SNAKE_CASE (`CRITICAL_PATTERNS`, `MAX_RETRIES`)
- Python: snake_case (`find_project_root`, `security_guard`)

## Error Handling
- Pattern: Try-catch with specific error types, logging to stderr
- Python hooks: Return JSON `{"decision": "block/continue", "reason": "..."}`
- TypeScript: Throw typed errors with descriptive messages

## Import Style
- ESM (`import/export`) for TypeScript/JavaScript
- Python standard imports
- Relative imports within same skill/module
- Type-only imports: `import type { ... }`

## Security Patterns
- Auth: Hook-based validation (PreToolUse hooks)
- Validation: Pattern matching against CRITICAL_PATTERNS
- Secrets: Never commit `.env`, credentials, API keys

## Architecture Patterns
- Skills: Self-contained directories with SKILL.md + implementation
- Agents: Single .md files with specialized prompts
- Commands: Single .md files with slash command definitions
- Hooks: Python/Shell scripts with JSON stdin/stdout protocol
