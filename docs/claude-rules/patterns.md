# Patterns

## Naming Conventions
- Files: kebab-case (`session-start.py`, `git-commit.md`)
- Functions: snake_case (`validate_skill`, `is_server_ready`)
- Classes: PascalCase (when used)
- Constants: UPPER_SNAKE_CASE

## Error Handling
- Pattern: try/except with graceful degradation
- Philosophy: Return None or fallback instead of raising (allows hooks to fail silently)
- Location: Each hook handles its own errors

## Import Style
- Python: Standard library first, then third-party
- Pattern: `import X` or `from X import Y`
- No relative imports in hooks

## Security Patterns
- Hooks validate tool calls before execution
- Security guard reviews dangerous operations
- No secrets in code (use environment variables)
