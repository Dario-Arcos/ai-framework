# Conventions

## Naming
- Files (skills): `kebab-case/` directories, `SKILL.md` entry point (uppercase)
- Files (agents): `kebab-case.md` (lowercase)
- Files (hooks): `kebab-case.py` or `kebab-case.sh`
- Files (scripts): `kebab-case.cjs`
- Shell functions: `snake_case` with module prefix (`ralph_log_info`, `ralph_log_warn`)
- Shell variables: `UPPER_SNAKE_CASE` for exports, `lower_snake_case` for locals
- JS functions: `camelCase` (`extractPreviousVersion`, `updateConfigJS`)

## Error Handling
- Shell: `set -u` (undefined variable errors), explicit exit codes
- Python: try/except with JSON error output for hook integration
- JS (sync-versions.cjs): try/catch with `process.exit(1)` on failure

## Imports
- JS: CommonJS `require()` in scripts (`.cjs` extension forces CJS in ESM project)
- Python: stdlib only (json, sys, os, subprocess) — no external dependencies in hooks
- Shell: source guards (`[[ -n "${_MODULE_LOADED:-}" ]] && return 0`)

## Code Style
- Markdown: YAML frontmatter (name, description) on all skills and agents
- Shell: include guards prevent double-sourcing of library modules
- Commits: conventional format `type(scope): description` (Spanish changelog, English commits)

## Versioning
- Semantic versioning via `npm version` → triggers sync-versions.cjs
- CHANGELOG.md: Keep a Changelog format in Spanish, validated on version bump
- Version synchronized across: package.json, plugin.json, README.md, VitePress config
